## Critical Blocker: Production Build Failure

The production build failure is the highest-priority item. Without a working build:
- No Lighthouse metrics can be captured
- No bundle sizes can be measured
- No interaction timing can be run
- The application cannot be deployed

**Root cause analysis:**
1. Dual lockfile issue: `pnpm-lock.yaml` exists at both repo root and `apps/web/`
2. Windows case-sensitivity: pnpm symlinks resolve with mixed C:/c: casing
3. Next.js workspace root inference picks up the wrong root
4. `vega-canvas` optional dependency `canvas` is unresolved (non-blocking)

**Suggested fix (for next session):**
- Remove the duplicate `apps/web/pnpm-lock.yaml` (keep only root)
- Or set `outputFileTracingRoot` in next.config.ts to resolve workspace root
- Add `canvas` to `next.config.ts` webpack externals to suppress the warning
